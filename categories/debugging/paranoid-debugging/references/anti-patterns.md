# Common Anti-Patterns & Red Flags

## Anti-Patterns

| Anti-Pattern | Why It's Wrong | Fix |
|--------------|----------------|-----|
| "这肯定是个小问题，直接修" | 未经验证的假设经常错 | 先建立复现，再谈修复 |
| "日志显示 X，所以是 Y" | 日志可能不完整/误导 | 用独立验证确认 |
| "修好了，手动测过了" | 无回归防护 | 必须添加/更新测试 |
| "测试太麻烦，相信我" | 违反证据驱动原则 | 无测试 = 未修复 |
| "顺便重构一下" | 扩大风险范围 | 最小修复，拒绝重构 |
| "这个边界不会发生" | 边界条件是 bug 温床 | 测试所有边界 |

## Red Flags - You're Doing It Wrong

\`\`\`
🚩 "我觉得问题应该是..."     → 先用证据证明
🚩 "这个我以前见过"          → 每个问题独立对待
🚩 "修复很简单"              → 简单不代表正确
🚩 "测试可以以后加"          → 无测试 = 未修复
🚩 "相信我，这次没问题"      → 证据 > 承诺
🚩 "这只是个临时方案"        → 临时方案会变成永久债
🚩 "不影响主流程"            → 边界问题最危险
\`\`\`

**All of these mean: Re-read the Strict Output Protocol.**
